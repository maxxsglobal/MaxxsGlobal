Database Setup
Connect to Supabase (free PostgreSQL database) — takes about 5 minutes

⚪ Database Not Connected

Enter your Supabase credentials below to connect. Once connected, all shipment data is stored in a real database and works across all devices worldwide.

🔑 Supabase Credentials
📋 Step-by-Step Setup Guide
1

Create a free Supabase account

Go to supabase.com → Sign up free → Create a new project → Choose a region close to your customers (e.g. Singapore for Middle East/Asia).

2

Create the shipments table

In your Supabase dashboard → go to SQL Editor → paste and run this SQL:

SQL to create table
CREATE TABLE shipments ( id BIGSERIAL PRIMARY KEY, tracking_id TEXT UNIQUE NOT NULL, status TEXT NOT NULL DEFAULT 'Pending', status_class TEXT NOT NULL DEFAULT 'status-pending', cargo TEXT NOT NULL, mode TEXT NOT NULL, eta TEXT NOT NULL, customer_name TEXT, route TEXT, timeline JSONB NOT NULL DEFAULT '[]', created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW() ); -- Allow public read (customers tracking shipments) ALTER TABLE shipments ENABLE ROW LEVEL SECURITY; CREATE POLICY "Public can read shipments" ON shipments FOR SELECT USING (true); -- Allow authenticated users to write (admin panel) CREATE POLICY "Service role can do everything" ON shipments FOR ALL USING (true); -- Auto-update updated_at CREATE OR REPLACE FUNCTION update_updated_at() RETURNS TRIGGER AS $$ BEGIN NEW.updated_at = NOW(); RETURN NEW; END; $$ LANGUAGE plpgsql; CREATE TRIGGER set_updated_at BEFORE UPDATE ON shipments FOR EACH ROW EXECUTE FUNCTION update_updated_at();
2b

Create the enquiries table (for the contact form)

In the same SQL Editor, also run this to enable enquiries from the website contact form:

SQL for enquiries table
CREATE TABLE IF NOT EXISTS enquiries ( id BIGSERIAL PRIMARY KEY, name TEXT NOT NULL, email TEXT NOT NULL, phone TEXT, message TEXT NOT NULL, status TEXT NOT NULL DEFAULT 'New', created_at TIMESTAMPTZ DEFAULT NOW() ); -- Allow anyone to submit a contact form ALTER TABLE enquiries ENABLE ROW LEVEL SECURITY; CREATE POLICY "Allow anon insert on enquiries" ON enquiries FOR INSERT TO anon WITH CHECK (true); -- Allow admin (service role / anon key) to read and update CREATE POLICY "Allow read enquiries" ON enquiries FOR SELECT USING (true); CREATE POLICY "Allow update enquiries" ON enquiries FOR UPDATE USING (true); CREATE POLICY "Allow delete enquiries" ON enquiries FOR DELETE USING (true);
3

Get your API credentials

In Supabase → Project Settings → API → copy:
• Project URL (e.g. https://abcdef.supabase.co)
• anon public key (the long JWT token)

4

Paste credentials above and click "Test & Save Connection"

The admin panel and customer website will then use your live database automatically.

5

Update the customer website file

Open maxxs-global-website.html → find SUPABASE_URL and SUPABASE_ANON_KEY → paste the same values. That's it — both files now share the same live database.
